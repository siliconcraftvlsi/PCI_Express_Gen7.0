# PCI Express Gen7 Controller Coverage Plan

Generated: 2026-05-21

## Functional Coverage Points

- Reset release and idle state reached.
- Main datapath valid/ready or equivalent traffic activity observed.
- Configuration or register writes and reads for each documented control/status region.
- Error or exceptional status events where the design exposes them.
- Feature-enable combinations for optional blocks.
- Boundary conditions: empty/full, min/max size, timeout, lock/unlock, corrected/uncorrected event, or DMA completion depending on project scope.

## Code Coverage Targets

| Metric | Target before release | Notes |
| --- | --- | --- |
| Line coverage | 80 percent or justified waiver | Use simulator-supported coverage where available. |
| Branch coverage | 70 percent or justified waiver | Review dead code and unreachable parameter branches. |
| Toggle coverage | Reviewed for major datapaths | Especially important for high-width datapaths. |
| Functional coverage | 100 percent for release-critical coverpoints | Partial features must remain documented as partial. |

## Coverage Reporting

Coverage databases are simulator-specific and are not generated by the current open-source iverilog flow. When using a commercial simulator, store coverage summaries under `coverage/reports/` and link them from `docs/signoff_checklist.md`.
